This is the Access Control List of the parent resource. An ACL entry is composed by resource + verb + subject, indicates a permission (verb) that can be granted or revoked to a subject over a resource.

The resources that have ACLs in Kii Platform are applications, users, groups, buckets, objects and topics, for each of them applies different verbs. A subject can be either a user or a group.

Verbs on applications, users and groups:
  • CREATE_NEW_BUCKET: Create a new bucket in the application, user or group.
  • CREATE_NEW_TOPIC: Create a new topic in the application, user or group.
Verbs on buckets:
  • QUERY_OBJECTS_IN_BUCKET: Perform a query in the bucket.
  • CREATE_OBJECTS_IN_BUCKET: Create an object in the bucket.
  • DROP_BUCKET_WITH_ALL_CONTENT: Delete the bucket with all the contained objects.
Verbs on objects:
  • READ_EXISTING_OBJECT: Read the content of the object (JSON data and body)
  • WRITE_EXISTING_OBJECT: Modify the content of the object or delete it.
Verbs on topics:
  • SUBSCRIBE_TO_TOPIC: Subscribe oneself to the topic to receive the messages sended to it.
  • SEND_MESSAGE_TO_TOPIC: Send a message to the topic.

ACLs in Kii Platform consist on while lists, so, permissions are rejected by default, to perform the action is needed to have the verb granted.

GET

/apps/{appID}/users/{userID}/buckets/{bucketID}/acl

Description

Retrieve the complete ACL of the resource.

Security

The administrator always can perform this action and depending on the resource can also do it:

  • User: the user.
  • Group: the group owner.
  • Bucket: the scope-owner and the bucket creator.
  • Object: the scope-owner and the object owner.
  • Topic: the scope-owner and the topic creator.
