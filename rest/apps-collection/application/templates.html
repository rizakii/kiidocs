Resource to manage custom templates. This is used to define custom messages to send message to all users when they use your application, per example in a verification email case.

You can use a place holders in the content message:
  • ${url} : It's replaced by the URL in some templates that need the user to click in a link (eg:email verification templates).
  • ${app.appID} : AppID of the application
  • ${user.displayName} : Display name of the user involved
  • ${user.userID} : UserID of the user involved
  • ${user.emailAddress} : Verified email address of the user involved. Empty if the email address is not set or is not verified yet.
  • ${user.phoneNumber} : Verified phone number of the user involved. Empty if the phone number is not set or is not verified yet.
  • ${user.unverifiedEmailAddress} : Unverified email address of the user involved. Empty if the email address is not set or is already verified.
  • ${user.unverifiedPhoneNumber} : Unverified phone number of the user involved. Empty if the phone number is not set or is already verified.
  • ${user.loginName} : Login name of the user involved, if it has.
  • ${user.phoneVerificationCode} : Verification code of the phone number, if there is an unverified phone number in the user.
  • ${user.emailVerificationCode} : Verification code of the email address, if there is an unverified email address in the user.
  • ${password} : In some templates which involve password handling, it's replaced by the password handled (eg: Password reset).
Location/apps/{appID}/templates
