This resource provides oauth token management
Location/oauth2/token
  • POST

Request Headers (Applies to all methods)

X-Kii-AppID
header
Required: Yes
X-Kii-AppKey
header
Required: Yes

POST

/oauth2/token

Description

Request a new token. To get a token with administrator credentials it must be provided the "client_id" and "client_secret". To get a token with user credentials it must be provided the "username" and "password" of the user.

Content-Typeapplication/vnd.kii.OauthTokenRequest+json or application/json

All the required information to request a token

Headers
Name Required? type Description
X-Kii-AppIDYesxml:stringThe AppID assigned by Kii for the current app
X-Kii-AppKeyYesxml:stringThe AppKey assigned by Kii for the current app
Params
Name Required? type Description
usernamestringThe user name one of the following values: - The login name of the user - The email address of the user - The email address of the user with format: EMAIL:{emailAddress} - The phone number of the user in international format: +XXX... - The phone number of the user in local format: PHONE:{country}/{localPhone} To use phone number or email they must be verified.
passwordstringThe password of the user
expiresAtlong The requested expiry time for this token. Milliseconds since standard epoch of 1/1/1970
client_idstringThe provided client id for the app admin
client_secretstringThe provided client secret for the app admin
Sample Request
{ "username": "[string]", "password": "[string]", "expiresAt": "[long]", "client_id": "[string]", "client_secret": "[string]" }
